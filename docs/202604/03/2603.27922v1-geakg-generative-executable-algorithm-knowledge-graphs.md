## 摘要
在算法求解问题的背景下，过程性知识——即算法设计和算子组合的诀窍——仍然隐含在代码中，在不同运行之间丢失，并且必须针对每个新领域重新进行工程设计。知识图谱（KGs）已被证明在组织陈述性知识方面非常有效，但目前的知识图谱范式在将过程性知识表示为可执行、可学习的图结构方面提供的支持有限。我们引入了生成式可执行算法知识图谱（GEAKG），这是一类知识图谱，其节点存储可执行算子，其边编码学习到的组合模式，且其遍历过程可生成解决方案。GEAKG 具有生成性（拓扑和算子由大语言模型合成）、可执行性（每个节点都是可运行的代码）和可迁移性（学习到的模式可以跨领域零样本泛化）。该框架在引擎层是领域无关的：相同的三层架构和基于蚁群算法（ACO）的学习引擎可以在不同领域实例化，并通过可插拔的本体（RoleSchema）进行参数化。两个不共享任何特定领域框架代码的案例研究为该框架假设提供了具体证据：(1) 在两个表格基准测试的 70 个跨数据集迁移对上进行神经架构搜索；(2) 组合优化，其中在旅行商问题上学习到的知识可以零样本迁移到调度和分配领域。综上所述，结果表明算法专业知识可以作为可执行知识图谱被显式表示、学习和迁移。

## Abstract
In the context of algorithms for problem solving, procedural knowledge -- the know-how of algorithm design and operator composition -- remains implicit in code, lost between runs, and must be re-engineered for each new domain. Knowledge graphs (KGs) have proven effective for organizing declarative knowledge, yet current KG paradigms provide limited support for representing procedural knowledge as executable, learnable graph structures. We introduce \textit{Generative Executable Algorithm Knowledge Graphs} (GEAKG), a class of KGs whose nodes store executable operators, whose edges encode learned composition patterns, and whose traversal generates solutions. A GEAKG is \emph{generative} (topology and operators are synthesized by a Large Language Model), \emph{executable} (every node is runnable code), and \emph{transferable} (learned patterns generalize zero-shot across domains). The framework is domain-agnostic at the engine level: the same three-layer architecture and Ant Colony Optimization (ACO)-based learning engine can be instantiated across domains, parameterized by a pluggable ontology (\texttt{RoleSchema}). Two case studies -- sharing no domain-specific framework code -- provide concrete evidence for this framework hypothesis: (1)~Neural Architecture Search across 70 cross-dataset transfer pairs on two tabular benchmarks, and (2)~Combinatorial Optimization, where knowledge learned on the Traveling Salesman Problem transfers zero-shot to scheduling and assignment domains. Taken together, the results support that algorithmic expertise can be explicitly represented, learned, and transferred as executable knowledge graphs.

#### 1. 核心问题与研究动机
论文关注的是算法设计中的**过程性知识（Procedural Knowledge）**——即“如何组合算子以解决问题”的诀窍。目前，这类知识通常隐含在代码中，随程序运行结束而丢失，且在面对新领域时需要重新工程化。传统的知识图谱（KG）擅长处理陈述性知识（“是什么”），但在表示可执行、可学习且可迁移的算法逻辑方面存在局限。研究动机在于建立一种能够显式表示、自动生成并跨领域复用算法设计经验的框架。

#### 2. 方法论：核心思想与技术细节
论文提出了 **GEAKG（Generative Executable Algorithm Knowledge Graphs）** 框架，其核心是将算法流程建模为由抽象角色（Roles）组成的图结构。
*   **三层架构：**
    *   **L0（拓扑层）：** 由大语言模型（LLM）生成的图结构，定义了算子角色间的合法转换关系。
    *   **L1（算子层）：** 存储可执行代码。LLM 根据角色定义合成具体的 Python 算子，并通过 AFO（Always-From-Original）原则保持多样性。
    *   **L2（知识层）：** 通过蚁群优化（ACO）在离线阶段学习边权重（信息素）和符号规则，识别哪些算子序列最有效。
*   **RoleSchema（领域本体）：** 预定义一套抽象角色（如“局部搜索”、“扰动”），使引擎与具体领域解耦。
*   **符号执行器（Symbolic Executor）：** 在在线部署阶段，无需调用 LLM，直接根据 L2 学习到的信息素和规则遍历图谱，生成并执行算法。

#### 6. 主要结论与发现
*   **跨领域迁移能力：** 在 TSP 上学到的搜索策略能有效提升 JSSP 和 QAP 的求解质量，证明了元级算法知识的通用性。
*   **零样本/零成本部署：** 训练好的 GEAKG 快照（仅 1-3 KB）在目标领域部署时无需重新训练或调用 LLM，实现了真正的知识持久化。

#### 7. 优点与亮点
*   **解耦设计：** 通过 RoleSchema 实现了算法逻辑与领域实现的彻底分离，同一套引擎可处理完全不同的问题。
*   **知识可视化：** 学习到的信息素矩阵和符号规则具有可解释性，能够回答“为什么这个算子序列更好”。
*   **高效性：** 将昂贵的 LLM 生成过程限制在离线阶段，在线阶段利用符号推理实现极速响应。

#### 8. 不足与局限
*   **人工建模依赖：** RoleSchema 仍需专家手动设计（约需数小时），尚未实现完全自动化的本体构建。
*   **搜索空间限制：** 目前主要针对排列类（Permutation）组合优化问题，尚未验证其在连续空间或混合整数规划中的表现。
*   **离线学习局限：** 符号执行器在运行过程中是静态的，缺乏根据实时反馈进行在线自适应调整的能力。
*   **基准对比：** 在某些特定领域，其性能虽优于通用启发式，但可能仍不及经过数十年人工精调的最先进（SOTA）专用求解器。
