## 摘要
智能体知识图谱问答（KGQA）要求智能体与知识图谱（KGs）进行迭代交互，这在训练数据稀缺性和推理泛化性方面都带来了挑战。具体而言，现有方法通常限制了智能体的探索：基于提示（prompting）的方法缺乏自主导航训练，而目前的训练流水线通常将推理局限于预定义的轨迹。为此，本文提出了 GraphWalker，这是一个新颖的智能体 KGQA 框架，通过自动化轨迹合成和分阶段微调来解决这些挑战。GraphWalker 采用两阶段有监督微调（SFT）训练范式：首先，在从受限随机游走路径合成的结构多样化轨迹上训练智能体，从而在知识图谱上建立广泛的探索先验；其次，在少量专家轨迹上进一步微调智能体，以培养其反思和错误恢复能力。大量实验表明，我们的分阶段 SFT 范式为轻量级强化学习（RL）阶段解锁了更高的性能上限，使 GraphWalker 在 CWQ 和 WebQSP 上达到了最先进的性能。在 GrailQA 和我们构建的 GraphWalkerBench 上的额外结果证实，GraphWalker 增强了对分布外推理路径的泛化能力。代码已在 https://github.com/XuShuwenn/GraphWalker 公开。

## Abstract
Agentic knowledge graph question answering (KGQA) requires an agent to iteratively interact with knowledge graphs (KGs), posing challenges in both training data scarcity and reasoning generalization. Specifically, existing approaches often restrict agent exploration: prompting-based methods lack autonomous navigation training, while current training pipelines usually confine reasoning to predefined trajectories. To this end, this paper proposes \textit{GraphWalker}, a novel agentic KGQA framework that addresses these challenges through \textit{Automated Trajectory Synthesis} and \textit{Stage-wise Fine-tuning}. GraphWalker adopts a two-stage SFT training paradigm: First, the agent is trained on structurally diverse trajectories synthesized from constrained random-walk paths, establishing a broad exploration prior over the KG; Second, the agent is further fine-tuned on a small set of expert trajectories to develop reflection and error recovery capabilities. Extensive experiments demonstrate that our stage-wise SFT paradigm unlocks a higher performance ceiling for a lightweight reinforcement learning (RL) stage, enabling GraphWalker to achieve state-of-the-art performance on CWQ and WebQSP. Additional results on GrailQA and our constructed GraphWalkerBench confirm that GraphWalker enhances generalization to out-of-distribution reasoning paths. The code is publicly available at https://github.com/XuShuwenn/GraphWalker

*   **核心问题**：现有的智能体知识图谱问答（Agentic KGQA）面临**训练数据稀缺**和**推理泛化性差**两大瓶颈。
*   **研究动机**：
    *   **提示词方法（Prompting）**：依赖大模型（LLM）的固有能力，缺乏在复杂、多噪声的全局知识图谱（KG）中自主导航的专门训练。
    *   **微调方法（Fine-tuning）**：通常局限于预定义的简单路径，导致智能体在面对未见过的复杂推理结构时表现脆弱。
    *   **强化学习（RL）**：如果没有良好的有监督微调（SFT）作为先验，RL 很难在广阔的图空间中发现有效的奖励路径。
*   **整体含义**：GraphWalker 旨在通过一种“课程学习”式的训练范式，让轻量级模型也能在全局知识图谱上实现高效的自主探索、反思与纠错。

#### 2. 方法论：核心思想与关键技术
GraphWalker 提出了一个包含**自动化轨迹合成**和**三阶段训练**的框架：

*   **数据构造流水线**：
    *   **GraphSynth-15k**：利用**受限随机游走（CRW）**在 KG 上生成 1.5 万条结构多样的路径（包含链式、汇聚式等），并利用大模型（如 Gemini/GPT-4o-mini）将其转化为自然语言问题和推理轨迹。这为智能体提供了广泛的探索先验。
    *   **GraphRoll-6k**：通过对复杂数据集（CWQ）进行拒绝采样，筛选出 6000 条包含**反思、回溯和错误恢复**的高质量专家轨迹。
*   **三阶段训练范式**：
    1.  **SFT 阶段 1（探索先验）**：在 GraphSynth 上训练，使模型掌握基本的图导航和工具调用规范。
    2.  **SFT 阶段 2（反思纠错）**：在 GraphRoll 上微调，培养模型在遇到死胡同时进行自我修正的能力。
    3.  **RL 阶段（性能飞跃）**：使用 **GRPO（群组相对策略优化）** 算法，配合简单的精确匹配（EM）奖励，进一步提升模型在长程决策中的上限。

*   **性能领先**：GraphWalker 在 CWQ 和 WebQSP 上均刷新了 SOTA。例如，GraphWalker-7B 在 CWQ 上的 EM 达到 79.6%，显著超过了使用 GPT-4 的方法。
*   **课程学习的价值**：先学“广度探索”（Stage 1）再学“深度反思”（Stage 2）的顺序至关重要，混合训练的效果远不如分阶段训练。
*   **RL 的催化作用**：两阶段 SFT 建立的稳健先验为 RL 提供了更高的起点，使其能突破 SFT 的表现天花板。

#### 7. 优点
*   **数据驱动的创新**：通过 CRW 算法系统性地解决了 KGQA 训练数据不足的问题，不再依赖昂贵的人工标注。
*   **训练策略优雅**：分阶段 SFT + 轻量级 RL 的组合，既保证了模型的稳定性，又挖掘了推理潜力。
*   **实战导向**：坚持在全局 KG 上进行实验，比在小子图上实验更符合真实应用场景。

#### 8. 不足与局限
*   **推理成本**：作为 Agent 框架，需要多次调用 LLM 和查询数据库，推理延迟和 API 成本高于单次生成的模型。
*   **对教师模型的依赖**：合成轨迹的质量高度依赖于 GPT-4o-mini 等强力模型的逻辑生成能力。
*   **知识图谱局限性**：实验主要基于 Freebase，对于其他领域（如医疗、金融）或多模态知识图谱的适应性尚待验证。
*   **实体链接依赖**：实验假设已知 Topic Entities，但在完全端到端的场景下，实体链接的错误可能会传播并影响最终效果。
