## 摘要
智能体知识图谱问答（KGQA）要求智能体与知识图谱（KGs）进行迭代交互，这在训练数据稀缺性和推理泛化性方面都带来了挑战。具体而言，现有方法往往限制了智能体的探索：基于提示（prompting）的方法缺乏自主导航训练，而目前的训练流程通常将推理局限于预定义的轨迹。为此，本文提出了 GraphWalker，这是一个新颖的智能体 KGQA 框架，通过“自动化轨迹合成”和“分阶段微调”来解决这些挑战。GraphWalker 采用两阶段有监督微调（SFT）训练范式：首先，在由受限随机游走路径合成的结构多样化轨迹上训练智能体，从而在知识图谱上建立广泛的探索先验；其次，在少量专家轨迹上进一步微调智能体，以培养其反思和错误恢复能力。大量实验表明，我们的分阶段 SFT 范式为轻量级强化学习（RL）阶段解锁了更高的性能上限，使 GraphWalker 在 CWQ 和 WebQSP 数据集上达到了最先进的性能。在 GrailQA 以及我们构建的 GraphWalkerBench 上的额外结果证实，GraphWalker 增强了对分布外推理路径的泛化能力。代码已在 https://github.com/XuShuwenn/GraphWalker 公开。

## Abstract
Agentic knowledge graph question answering (KGQA) requires an agent to iteratively interact with knowledge graphs (KGs), posing challenges in both training data scarcity and reasoning generalization. Specifically, existing approaches often restrict agent exploration: prompting-based methods lack autonomous navigation training, while current training pipelines usually confine reasoning to predefined trajectories. To this end, this paper proposes \textit{GraphWalker}, a novel agentic KGQA framework that addresses these challenges through \textit{Automated Trajectory Synthesis} and \textit{Stage-wise Fine-tuning}. GraphWalker adopts a two-stage SFT training paradigm: First, the agent is trained on structurally diverse trajectories synthesized from constrained random-walk paths, establishing a broad exploration prior over the KG; Second, the agent is further fine-tuned on a small set of expert trajectories to develop reflection and error recovery capabilities. Extensive experiments demonstrate that our stage-wise SFT paradigm unlocks a higher performance ceiling for a lightweight reinforcement learning (RL) stage, enabling GraphWalker to achieve state-of-the-art performance on CWQ and WebQSP. Additional results on GrailQA and our constructed GraphWalkerBench confirm that GraphWalker enhances generalization to out-of-distribution reasoning paths. The code is publicly available at https://github.com/XuShuwenn/GraphWalker

#### 1. 核心问题与整体含义（研究动机和背景）
知识图谱问答（KGQA）的核心挑战在于如何在海量且有噪声的结构化数据中进行多步推理。现有的方法主要分为检索式（IR）和解析式（SP），但它们通常局限于静态、预定义的搜索范围。新兴的“智能体（Agentic）”方法允许模型与图谱交互，但面临两大瓶颈：
*   **训练数据稀缺**：高质量的智能体交互轨迹（包含思考、行动、观察）难以获取。
*   **泛化能力差**：现有模型往往只能模仿预定义的简单路径，缺乏在复杂、未知图谱结构中自主导航、反思和纠错的能力。
**GraphWalker** 的动机是通过自动化合成多样化的交互轨迹，构建一个从“广泛探索”到“精细反思”的课程学习路径，从而提升轻量级模型在全局知识图谱上的推理上限。

#### 2. 方法论：核心思想与关键技术
GraphWalker 提出了一个**两阶段有监督微调（SFT）+ 强化学习（RL）**的训练范式：

*   **数据构造阶段**：
    *   **GraphSynth-15k**：利用**受限随机游走（CRW）**在知识图谱上生成 1.5 万条结构多样的路径（如 2-5 跳的链状、汇聚状等），并利用 LLM 将路径转化为自然语言问题和思考轨迹。目的是建立强大的“探索先验”。
    *   **GraphRoll-6k**：通过拒绝采样获取 6000 条专家级轨迹，重点包含模型在遇到死胡同时的“反思”和“回溯”行为。
*   **训练阶段**：
    *   **SFT 阶段 1**：在 GraphSynth 上训练，让模型学会基本的图谱导航和工具调用。
    *   **SFT 阶段 2**：在 GraphRoll 上微调，强化模型在复杂环境下的纠错和自我修复能力。
    *   **RL 阶段**：使用 **GRPO（群体相对策略优化）**算法，以最终答案的准确性（Exact Match）作为稀疏奖励，进一步突破 SFT 的性能天花板。
*   **关键技术细节**：智能体配备了 `get_relations` 和 `get_triples` 两个核心工具，并引入了基于 LLM 的关系重排序（Reranking）来过滤冗余信息。

*   **数据集**：核心基准为 **CWQ**（复杂组合问答）和 **WebQSP**（简单事实问答）。此外，在 **GrailQA**（零样本测试）和自建的 **GraphWalkerBench**（包含训练集中未见的复杂结构）上验证泛化性。

*   **实验设置**：所有实验均在**全局知识图谱（Global KG）**环境下进行，而非预先提取的小型子图，这更符合真实应用场景。

*   **硬件**：使用了 **8 台 NVIDIA A100 (80GB) GPU**。
*   **训练细节**：
    *   SFT 阶段：Batch size 32，学习率 1e-5，训练 2 个 epoch。
    *   RL 阶段：使用 GRPO 算法，训练 100 个 step，Global batch size 为 512。
*   **模型底座**：主要基于 Qwen2.5 (3B/7B) 和 Llama-3.1 (8B)。

#### 5. 实验数量与充分性
*   **实验规模**：论文进行了主实验对比、跨任务迁移实验（CWQ 训练，WebQSP 零样本测试）、OOD 泛化实验（GrailQA/GraphWalkerBench）。
*   **消融实验**：非常详尽，涵盖了 SFT 两个阶段的必要性、训练顺序的影响、数据规模的缩放效应（Scaling Law）、以及关系重排序的效果。

#### 6. 主要结论与发现
*   **性能领先**：GraphWalker-7B 在 CWQ 和 WebQSP 上均刷新了 SOTA 记录，甚至超越了使用 GPT-4 的提示词方法。
*   **课程学习的必要性**：先学“探索”再学“反思”的阶段式训练显著优于混合数据的一次性训练。
*   **RL 的催化作用**：两阶段 SFT 建立的强大先验是 RL 成功的关键；没有 SFT 阶段 1 的探索能力，RL 很难在稀疏奖励下收敛。
*   **强泛化性**：模型能够处理训练中从未见过的复杂图结构（如 IP/PI 结构）。

#### 7. 优点
*   **数据驱动的创新**：通过受限随机游走解决了智能体训练数据匮乏的问题，且数据构造流程高度自动化。
*   **训练范式优雅**：将复杂的智能体行为拆解为探索、反思、对齐三个逻辑清晰的阶段。
*   **实战导向**：坚持在全局 KG 上运行，证明了轻量级模型（3B/7B）通过合理训练可以具备极强的自主导航能力。

#### 8. 不足与局限
*   **推理成本**：作为智能体方法，需要多次调用 LLM 和查询数据库，推理延迟高于单次生成的解析式方法。
